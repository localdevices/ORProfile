API reference#

The API of orprofile consists of several classes and functions. Here we document these in a structured way so that you can reuse the code, build your own scripts, notebooks or even software around it.

Note

We highly recommend to use the excellent xarray manual side-by-side with orprofile to understand how to effectively work with the xarray ecosystem.

In the remaining sections, we describe the API classes, and the functions they are based on.

Mesh class#

api.Mesh(spline_shape[, n, m, points])

api.Mesh.points

returns: loaded sonar survey points :rtype: gpd.GeoDataFrame

api.Mesh.splines

returns: 4 splines defining the area of interest (2 along banks and 2 perpendicular to river :rtype: gpd.GeoDataFrame

api.Mesh.mesh_kernel

returns: contains the curvilinear grid based on defined splines and row/column parameters :rtype: meshkernel.MeshKernel

api.Mesh.mesh2d

rtype: xu compatible 2D grid

api.Mesh.plot([ax, tiles, extent, ...])

Plot available data in a geographically aware plot (cartopy)

api.Mesh.map_rowcol_wise(func[, name, rowcol])

Map a 1D function over each row in the defined mesh object under self.mesh2d.

api.Mesh.read_points(fn[, crs])

Read a set of surveyed points.

Depth functions#

api.depth_2d(mesh, alpha, L, kappa, c, A)

Lacey, solved 2-dimensionally, wrapped in xugrid data model

profile.depth_profile_left_right_parameters(s, ...)

Savenije eq.

profile.depth_from_width(s[, phi, h_m])

Basic implementation of Eq.

profile.deepest_point_y_dist(x, alpha, L, kappa)

Estimates the location of the deepest point as offset from the centerline as function of one or more longitudinal coordinates x, assuming a sinusoidal function.
